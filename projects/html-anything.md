# html-anything

## 一句话定位
Agentic HTML 编辑器 — 本地 AI Agent 生成 HTML 内容，75 种技能 × 9 种输出形式，零 API Key。

## 它解决的问题
内容创作者需要快速生成多种格式的内容（杂志、PPT、海报、推文、原型图等），但现有工具要么需要 API Key 配置，要么输出格式单一。html-anything 让本地 AI Agent 直接生成可用 HTML。

## 为什么值得关注（2026-05-17）
- 2,425 stars，增速快
- BYOK（自带 Key）模式 + 零配置，降低使用门槛
- 支持 Claude Code / Cursor / Codex / Gemini / Copilot 等多种 Agent
- 75 Skills × 9 Surfaces 的矩阵设计覆盖面广

## 热度来源判断
- 内容生成是 AI 的刚需场景
- 多 Agent 兼容性降低了使用门槛
- 泡沫风险中等 — 很多 Skill 可能是模板包装

## 关键技术亮点
1. **BYOK 零 API Key 设计** — 用户自带 Key，平台不收费
2. **75 Skills × 9 Surfaces 矩阵** — 每种内容类型对应多种输出形式
3. **沙箱化预览** — 安全的 HTML 渲染环境
4. **一键发布** — 直接推送到微信/X/知乎等平台

## 架构启发
- Skill 矩阵设计是 Agent 内容生成的有效组织方式
- BYOK 模式避免了平台的 API 成本

## 定位判断
工具型产品，适合个人和小团队使用。不是基础设施或平台。

## 风险 / 局限 / 泡沫点
1. **75 Skills 质量参差不齐** — 可能很多是简单模板
2. **依赖本地 Agent 的能力** — 效果受限于底层 LLM
3. **长期可持续性存疑** — 纯 Skill 项目维护成本高

## 与同类项目的关系
- vs **v0.dev**：v0 更偏 UI 组件，html-anything 更偏内容
- vs **Gamma.app**：Gamma 是 SaaS，html-anything 是本地 Agent Skill

## 是否值得持续跟踪
**低优先级。** 工具型项目，关注 Skill 质量和用户反馈。

## 后续观察点
1. 75 Skills 中有多少是高质量模板
2. 社区是否贡献新的 Surface 和 Skill
3. 实际用户的内容生成质量反馈
