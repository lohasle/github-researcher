核心设计：凭据边界（Credential Boundaries） Agent 永远拿不到原始 API Key。iron-proxy 在出站请求时替换占位符为真实凭据，只替换到指定 host + header。这意味着 Agent 可以调用服务，但不能泄露密钥。
