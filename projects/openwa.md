# OpenWA

## 一句话定位
免费、开源、自托管的 WhatsApp API 网关，替代官方商业 API。

## 它解决的问题
WhatsApp Business API 官方接入门槛高、费用贵，且依赖 Meta 平台。OpenWA 提供自托管方案，适合需要 WhatsApp 集成但预算有限或需要数据主权控制的团队。

## 为什么值得关注（2026-05-29）
1. 6.9K stars，本周 +2.1K，增速较快
2. 自托管、免费，解决 WhatsApp API 接入的门槛问题
3. TypeScript 实现，部署方便

## 热度来源判断
- WhatsApp 是全球最大即时通讯平台，API 需求真实
- 自托管方案对中小企业和发展中国家市场有吸引力
- 但存在 WhatsApp 政策风险（可能被封禁）

## 关键技术亮点
1. 自托管 API 网关，完全控制数据
2. TypeScript 实现，与 Node.js 生态集成
3. 提供标准 REST API 接口

## 架构启发
- 自托管消息网关是企业通讯基础设施的重要组件
- API 网关模式可以复制到其他消息平台

## 定位判断
- **工具型**，非基础设施
- 依赖 WhatsApp 平台政策，存在不确定性

## 风险/局限/泡沫点
1. **WhatsApp 政策风险**：Meta 可能封禁非官方 API
2. 稳定性依赖 WhatsApp Web 的逆向工程
3. 非官方 API，没有 SLA 保障
4. 不适合企业级生产环境

## 是否值得持续跟踪
**观察型。** 需求真实但风险较高，适合轻量观察。

## 后续观察点
1. WhatsApp 是否会调整 API 政策
2. 项目稳定性和社区活跃度
3. 是否有企业采用案例
