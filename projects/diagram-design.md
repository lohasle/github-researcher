# Diagram Design

## 一句话定位
13 种编辑级图表类型的 Claude Code Skill，纯 HTML + SVG 实现，无阴影、无 Mermaid 模板痕迹。

## 它解决的问题
目标用户是使用 Claude Code 的开发者和写作者。痛点是：Mermaid 图表丑且难以定制，手动画 SVG 成本高，需要一种"告诉 AI 要什么图就出什么图"的方案。

## 为什么值得关注（2026-04-22）
代表了 Claude Code Skill 从"通用工具"到"专业设计"的细分趋势。13 种图表类型的覆盖面超越了一般的图表工具。

## 热度来源判断
Claude Code 生态热度带动。1300+ star 在 Skill 项目中属于中上水平。真实需求——技术文档中的图表质量长期被忽视。

## 关键技术亮点
1. **13 种编辑级图表**：流程图、时间线、金字塔、矩阵等，覆盖常见编辑需求
2. **纯 HTML + SVG**：无外部依赖，输出干净可直接嵌入文档
3. **设计规范严格**：无阴影、无渐变，坚持编辑级美学

## 架构启发
- Skill 作为"设计品味注入器"：不只生成图表，还定义了什么是"好的图表"
- AI + 模板的结合方式：预定义高质量模板，AI 负责内容填充

## 定位判断
Claude Code Skill 生态中的专业图表工具。工具型，生命周期取决于 Skill 生态整体命运。

## 风险 / 局限 / 泡沫点
1. Claude Code 可能内置更好的图表生成能力
2. 13 种类型可能不够用也可能太多，维护负担
3. Skill 生态碎片化风险——每个设计需求一个 Skill

## 与同类项目的关系
- vs **Mermaid**：Mermaid 是通用图表语言，diagram-design 追求编辑级设计质量
- vs **huashu-design**：huashu 更偏通用设计 Skill，diagram-design 聚焦图表

## 是否值得持续跟踪
可选。关注 Skill 生态细分趋势，但单项目价值有限。

## 后续观察点
1. 图表类型是否持续扩展
2. 是否被其他 Skill 框架整合
3. 编辑级设计质量是否被用户认可
