# Kami

## 一句话定位
好内容配好排版——内容优先的纸张设计工具，用 HTML 实现高质量的文档/幻灯片排版。

## 它解决的问题
目标用户是需要输出高质量文档或排版内容的开发者和内容创作者。痛点是：Markdown 只能写内容，设计工具（Figma/InDesign）学习成本高，中间缺少一个"写了就是好排版"的工具。

## 为什么值得关注（2026-04-22）
tw93（Pake 作者）出品，1354 star / 2 天。代表了"开发者做设计"的趋势延续。与 Claude Code Skill 生态的设计工具方向形成呼应。

## 热度来源判断
tw93 个人品牌贡献了大部分早期流量。但项目本身确实解决了"内容即排版"的需求，不是纯流量泡沫。

## 关键技术亮点
1. **HTML-native 设计**：纯 HTML/CSS 实现，无需设计工具
2. **内容优先理念**：专注内容，排版自动化
3. **纸张隐喻**：用纸张而非屏幕作为设计画布

## 架构启发
- **内容与呈现分离**的简化版本——不是完整的 CSS 层分离，而是"写内容即得排版"
- 开发者友好设计工具的持续探索

## 定位判断
工具型产品，位于 Markdown 编辑器和专业设计工具之间。可能被 AI 生成工具整合。

## 风险 / 局限 / 泡沫点
1. tw93 项目一贯高开低走（Pake 后续维护力度待验证）
2. 单人项目，可持续性存疑
3. 可能被 AI 编程工具（Claude Code 等）直接替代

## 与同类项目的关系
- vs **Slidev**：Slidev 聚焦幻灯片，Kami 更偏文档排版
- vs **huashu-design**：huashu 是 Claude Code Skill，Kami 是独立工具

## 是否值得持续跟踪
可选。项目本身不复杂，关注点在于 AI 时代"设计工具"的演进方向。

## 后续观察点
1. 是否形成社区贡献
2. 与 AI Agent 工具链的集成方式
3. tw93 的持续维护投入
