# 项目档案结构规范

每个重点跟踪的项目必须有 `projects/{slug}.md` 档案。

## 结构规范

```markdown
---
title: "项目名"
slug: "项目slug"
date_added: "YYYY-MM-DD"
category: "分类"（基础设施候选/平台候选/工具型/生产可用/观察型）
emoji: "🚀"
stars: "N stars"
stars_delta: "增速描述"
language: "主要语言"
score: 72
tags: ["标签1", "标签2"]
url: "https://github.com/org/repo"
---

# 项目名

## 一句话定位
[一句话说清楚这个项目做什么]

## 它解决的问题
[目标用户是谁，痛点是什么]

## 为什么值得关注（YYYY-MM-DD）
[当日判断，为什么选为重点]

## 热度来源判断
[为什么火，是真实需求还是泡沫]

## 关键技术亮点
[至少 3 个技术点]

## 架构启发
[设计哲学、trade-offs]

## 定位判断
[在生态中的位置]

## 风险 / 局限 / 泡沫点
[至少 2 个风险]

## 与同类项目的关系
[与 2-3 个竞品对比]

## 是否值得持续跟踪
[明确的跟踪建议]

## 后续观察点
[至少 2 个具体观察点]

---
*首次记录：YYYY-MM-DD*
```

## 质量底线

- 文件大小 > 1000 bytes
- 包含所有上述章节（不允许空章节）
- score 是 0-100 的整数
- url 必须是有效的 GitHub 链接
- category 使用标准分类词汇
