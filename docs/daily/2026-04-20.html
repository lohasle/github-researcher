1. Self-Healing Browser Agent：从框架走向裸协议

browser-harness (2031⭐, 4/17 新建) 是 browser-use 团队的最新作品，但它不是 browser-use 的升级版，而是一个完全不同的范式。它放弃了 Playwright/Selenium 等 browser automation 框架，直接通过 CDP (Chrome DevTools Protocol) WebSocket 连接浏览器。核心创新点：agent 在运行时自己写缺失的辅助函数——当 helpers.py 里没有 upload_file() 时，agent 自动编辑文件添加这个函数，然后继续执行。

这个设计哲学很值得注意： - 没有框架层，没有 recipe，没有预定义流程 - 全部代码约 592 行 Python（run.py 36 行 + helpers.py 195 行 + admin.py/daemon.py 361 行） - Agent 自主生成 domain skills 并持久化到 domain-skills/ 目录 - 支持远程浏览器（免费 tier 3 个并发）

架构师判断：这是 Browser Use 领域从「框架封装」走向「agent-native 裸协议」的信号。对 CI/CD、自动化测试、RPA 场景有直接冲击。值得关注其 self-healing 机制是否能长期稳定运行。

4. 云原生 Postgres 平台开源化：Xata 的 Copy-on-Write Branching

xata (634⭐, 4/15 新建) 是 Xata 公司将其云原生 Postgres 平台开源。核心能力： - 基于 CloudNativePG + OpenEBS 构建 - Copy-on-Write storage branching（秒级复制 TB 级数据） - Scale-to-zero（空闲自动移除计算，连接时自动恢复） - Auto-scaling + bin-packing - 高可用 / 读写分离 / 自动 failover - Serverless driver（HTTP/WebSocket）

架构师判断：这是 基础设施候选。对需要 preview/staging 环境快速复制的团队非常有价值。CoW branching 在 K8s 上实现了类似 PlanetScale 的体验，但是基于 Postgres 而非 MySQL。对标：Neon（同样做 Postgres branching，但未开源 server 端）、Supabase（更偏 BaaS）。
